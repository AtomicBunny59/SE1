# Source Control

## Branching
- main → stable code
- feature branches:
  - feature/login-page
  - fix/navbar-bug

## Commit Rules
- Use clear messages:
  - Add login validation
  - Fix navbar bug
- Avoid vague commits:
  - "stuff"
  - "updates"

## Pull Request Process
1. Pull latest main
2. Create feature branch
3. Make changes
4. Test locally
5. Commit changes
6. Push branch
7. Open PR
8. Request review
9. Fix feedback
10. Merge after approval

## Rules
- No direct commits to main
- Use PRs for all changes
- Delete branches after merge
